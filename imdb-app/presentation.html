Database Tables Introduction

7 Core Tables with Normalized Design

1. app_user

Purpose: Demo users and administrators for the application

Column Type Description
user_id INT (PK) Unique user identifier (Auto-increment)
username VARCHAR(50) Username for login (UNIQUE, NOT NULL)
email VARCHAR(255) Email address (UNIQUE, NOT NULL)
is_admin TINYINT(1) Administrator flag (Default: 0)
created_at TIMESTAMP Account creation timestamp

Sample Data: 4 demo users (alice, bob, carol, demo_user)

2. title

Purpose: Movies and TV series metadata

Column Type Description
title_id INT (PK) Unique title identifier (Auto-increment)
imdb_tconst VARCHAR(12) IMDb title ID (UNIQUE, e.g., tt0000001)
primary_title TEXT Movie/series title (NOT NULL)
start_year INT Release year (CHECK: >= 1870)
title_type VARCHAR(20) Type: 'movie', 'tvSeries', 'tvMovie' (NOT NULL)
runtime_minutes INT Duration in minutes (CHECK: > 0)
avg_rating DECIMAL(3,1) IMDb average rating (0.0-10.0)
num_votes INT Number of IMDb votes (Default: 0)

Data: 264 titles loaded from IMDb dataset

Database Tables (Continued)

3. person

Purpose: Actors, directors, writers, and other industry professionals

Column Type Description
person_id INT (PK) Unique person identifier (Auto-increment)
imdb_nconst VARCHAR(12) IMDb name ID (UNIQUE, e.g., nm0000001)
primary_name TEXT Actor/director/writer name (NOT NULL)
birth_year INT Birth year (CHECK: >= 1850)
death_year INT Death year (CHECK: >= birth_year if specified)

Data: 4,990 people from IMDb dataset

4. genre_lookup

Purpose: Reference table for standardized genres

Column Type Description
genre_id INT (PK) Unique genre identifier (Auto-increment)
genre_name VARCHAR(50) Genre name (UNIQUE, NOT NULL)

Genres: 27 standard IMDb genres (Action, Adventure, Animation, Biography, Comedy, Crime, Documentary, Drama, Family, Fantasy, Film-Noir, Game-Show, History, Horror, Music, Musical, Mystery, News, Reality-TV, Romance, Sci-Fi, Short, Sport, Talk-Show, Thriller, War, Western)

5. title_genre

Purpose: Junction table mapping titles to genres (Many-to-Many relationship)

Column Type Description
title_id INT (PK, FK) Reference to title (ON DELETE CASCADE)
genre_id INT (PK, FK) Reference to genre_lookup (ON DELETE RESTRICT)

Data: 167 title-genre relationships loaded

Database Tables (Continued)

6. title_person_role

Purpose: Junction table mapping people to titles with their roles (Many-to-Many relationship)

Column Type Description
title_id INT (PK, FK) Reference to title (ON DELETE CASCADE)
person_id INT (PK, FK) Reference to person (ON DELETE CASCADE)
role_type VARCHAR(30) (PK) Role: 'actor', 'director', 'writer', 'producer'
characters TEXT Character names (for actors)

Data: 25 cast/crew entries loaded

7. user_rating

Purpose: User ratings and reviews for titles

Column Type Description
user_rating_id INT (PK) Unique rating identifier (Auto-increment)
user_id INT (FK) Reference to app_user (ON DELETE CASCADE)
title_id INT (FK) Reference to title (ON DELETE CASCADE)
rating_value INT User's rating 1-10 stars (CHECK: BETWEEN 1 AND 10)
review_text TEXT Optional review comment (max 500 chars)
rated_at TIMESTAMP When the rating was created (Default: CURRENT_TIMESTAMP)

Constraints: UNIQUE(user_id, title_id) - One rating per user per title

Data: 23 user ratings from 4 demo users

Database Design Summary

Key Features and Constraints

Design Principles

  • Normalization: Third Normal Form (3NF)
  • Junction Tables: Many-to-Many relationships properly modeled
  • Referential Integrity: 7 Foreign Key constraints
  • Data Integrity: 5 UNIQUE constraints, 3 CHECK constraints
  • Indexing: 5 indexes including 1 composite index

Constraints

  • UNIQUE: Username, email, IMDb IDs
  • CHECK: Rating (1-10), Year (>= 1870), Runtime (> 0)
  • FK CASCADE: Auto-delete related records
  • FK RESTRICT: Prevent genre deletion
  • One Rating Rule: UNIQUE(user_id, title_id)
